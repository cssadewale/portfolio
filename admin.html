Complete Feature Guide

Photos & Media: Go to Photo Manager to upload and assign photos to your hero, about page, and CV. Go to Media Library for all other images. Files are stored in browser storage until you export.

Changing Your Password: Go to Security & Password. Enter your current password, set a new one, and save. Download your admin.html file and re-upload to GitHub after changing the password.

SEO / Google Search: Go to SEO & Meta Tags. Fill in your page title, description, and keywords. Submit your sitemap at Google Search Console for faster indexing.

Section Visibility: Go to Show / Hide Sections to toggle any homepage section on or off without deleting content.

Deploy Changes: Always go to Export & Deploy after editing. Download portfolio-data.json and push to GitHub. Live in 60 seconds.

Contact Form Messages: Go to Messages. Make sure to set up Formspree (formspree.io) with your email and replace YOUR_FORMSPREE_ID in index.html with your actual endpoint ID.

Backup: Go to Backup & Restore and download a JSON backup before every major change. This protects your work.

GitHub Connection Settings
● Checking token...
The GitHub repo connected to your Cloudflare Pages site
Your token is stored in your browser only — never sent anywhere except GitHub directly.
How to get your token: 1. Go to github.com/settings/tokens → 2. Click "Generate new token (classic)" → 3. Give it a name (e.g. "Portfolio Admin") → 4. Check repo (full repository access) → 5. Click Generate → Copy the token → Paste here.
How It Works — Explained Simply
1
One-time setup (5 minutes): Create a GitHub Personal Access Token and paste it above. You only do this once. The token is saved in your browser.
2
Make your changes: Edit projects, story, certifications, photos, skills — anything in the admin panels. Changes are saved locally as you go.
3
Click "Push Changes Live": The admin sends your portfolio-data.json directly to GitHub via the GitHub API. No manual file upload needed.
4
Site rebuilds in 30–60 seconds: Cloudflare Pages detects the new commit and automatically rebuilds your site. Your live site is updated — no further action needed.
Important: Your portfolio pages must include <script src="portfolio-loader.js"></script> before the closing </body> tag. This is the script that reads the data file and applies your changes to each page.
Manual Alternative
If you prefer to deploy manually or the GitHub API is not connected, download the file and upload it yourself.
Then: Go to github.com/cssadewale → open your portfolio repo → upload portfolio-data.json → commit. Cloudflare Pages rebuilds automatically.
