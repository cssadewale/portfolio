About Token Security

Your GitHub token is stored in sessionStorage — it disappears when you close your browser tab. It is never pushed to GitHub and never stored in portfolio-data.json.

This means: each time you open admin.html on a new device or new tab, you enter your token once. This is intentional — it is the correct security model for a static site admin. The token is the key to your GitHub. It should not live anywhere permanently except your own memory or a password manager.

Your admin password (SHA-256 hash) is stored in portfolio-data.json on GitHub. This means it syncs across every device. When you change your password and push, it updates everywhere immediately.

This opens the full diagnostic tool that tests your sitemap, robots.txt, HTTPS, and domain consistency automatically.

Quick fix if sitemap says "Couldn't fetch":
1. Open https://cssadewale.pages.dev/sitemap.xml in your browser. If you see a 404 — the file was not uploaded.
2. Make sure sitemap.xml and robots.txt are in your GitHub repo root.
3. In Search Console, the field shows your domain — type ONLY sitemap.xml in the box, nothing else.
4. Even without sitemap: use URL Inspection Tool → request indexing for each page directly.

Quick Guide

Cross-device sync: All your content lives in portfolio-data.json on GitHub. Open admin.html on any device, enter your token, and you see the same data. Changes you push from your tablet are immediately available on your laptop.

Password security: Your password is a SHA-256 hash stored in GitHub — not plaintext, not localStorage. Change it here and push — it updates on every device instantly.

GitHub token: Your PAT is in sessionStorage only — cleared when you close the browser tab. Enter it once per session. Never stored in your repo.

Deploy workflow: Edit anything → click "Push Live" → done. 30–60 seconds to live.

Sitemap "Couldn't fetch": Go to the SEO Check panel. Most likely cause: file not uploaded to GitHub root. Second most likely: wrong URL typed in Search Console.
